# Synthetic Category Theory in CaTT

This repository contains a literate Agda formalization of the type theory CaTT
as infrastructure for synthetic category theory in weak omega-categories.
The long-term goal is to express categorical constructions inside CaTT itself,
so that higher coherence data is generated by the type theory rather than added
by hand afterward.

## What This Repository Is About

CaTT is a type theory for weak omega-categories. Its types describe cells:
- There is a base type, whose terms we think of as objects/0-cells
- For a type A and terms u,v:A, thought of as n-cells, there is a hom type
  u -->_A v, whose terms we think of as (n+1)-cells from u to v.
Terms are generated in two ways:
- Every variable gives a term
- Terms can be combined into various coherences, which are formulated using
   the notion of a pasting context.

We interpret 0-cells as categories, 1-cells as functors, 2-cells as natural
isomorphisms, and so on. The notions of composition, associativity, whiskering, 
etcetera  arise from the general CaTT coherences.

The goal is to use this basic setup to formulate the basic axioms from
https://drive.google.com/file/d/1lKaq7watGGl3xvjqw9qHjm6SDPFJ2-0o/view?usp=sharing
The advantage of using CaTT is that the basic axioms can be stated in a fully 
coherent way.

## Current Status

The current implementation is best viewed as a layered development.

| Path | Role | Status |
|---|---|---|
| `I-CaTT/0-Agda` | Small logical and boolean helper lemmas | stable |
| `I-CaTT/1-preCaTT` | Raw preCaTT syntax, substitution, dependency, pasting, fullness | implemented |
| `I-CaTT/2-CaTT` | Intrinsic CaTT syntax, wrappers, examples, typed functoriality machinery | active |
| `I-CaTT/3-Naturality` | Naturality roadmap/specification | planning document |
| `I-CaTT/4-Diagrams` | First diagram-telescope layer | stub |

More concretely:

- `1-preCaTT` contains the computational backbone: contexts, de Bruijn
  variables, types, terms, substitutions, dependency predicates, dimensions,
  pasting contexts, and fullness.
- `2-CaTT` packages that raw layer into intrinsically typed CaTT syntax with
  explicit typing and fullness certificates.
- `2c-Examples` constructs concrete coherence terms such as identity,
  composition, associativity, and right whiskering.
- `2e`, `2f`, and `2g` develop the current functoriality line. The recursive
  shape of the depth-0 construction is present, but some hard side conditions
  are still postulated.
- `3-Naturality` is currently a detailed plan for the next stage, not yet a
  working implementation.
- `4a-Diagrams` already defines diagram types as typed telescopes, independently
  of the later naturality-dependent hom-type construction.

## Getting Started

This project uses literate Agda files (`.lagda.md`). The repository ships the
library file [`catt-synthetic.agda-lib`](./catt-synthetic.agda-lib), which
includes the active source directories and depends on the Agda standard library.

Typical workflow:

1. Install Agda and the standard library.
2. Open the repository root in your editor.
3. Load individual modules directly, rather than expecting a single umbrella
   module for the whole project.

From the repository root, the following commands are representative:

```powershell
agda I-CaTT/1-preCaTT/1-preCaTT.lagda.md
agda I-CaTT/4-Diagrams/4-Diagrams.lagda.md

## What Is Not Finished Yet

Several important parts of the intended program are present only as plans or
partial implementations.

- There is not yet a single complete top-level module that re-exports the whole
  active development.
- The naturality construction is specified but not implemented.
- The full diagram layer, including hom-types between diagrams, depends on that
  naturality work.
- The later category, groupoid, and constructor branches are placeholders.
- Some current functoriality files still use postulates for technically hard
  typing and preservation lemmas.

That incompleteness is deliberate: the repository is a research codebase, and
the implemented parts are meant to make the current mathematical and engineering
state explicit rather than hide unfinished obligations.
